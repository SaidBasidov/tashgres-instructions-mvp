## Способ интеграции

Стабильные ID прежнего импорта сохранены. Для каждого документа лёгкая модель с метаданными и очищенным поисковым текстом остаётся в `src/data/importedDocuments`, а полный `document.json`, HTML, оригинал, PDF и изображения размещены в `public/documents/<rich-id>`.

Полная разметка загружается через `fetch` только при открытии документа. HTML, PDF и изображения не импортируются в JavaScript bundle. Пути формируются с учётом `import.meta.env.BASE_URL`. Скрипты, внешние ресурсы, формы, inline-обработчики и небезопасные URL удаляются перед вставкой HTML. Изображения загружаются лениво.

Таблицы отображаются настоящими HTML-таблицами с сохранением `rowspan`/`colspan`; на экране прокрутка остаётся внутри таблицы, а в печати таблица адаптируется к ширине листа. На мобильной ширине 390 px общий горизонтальный скролл отсутствует, при этом широкая таблица сохраняет собственную прокрутку.

Навигация строится по подготовленному массиву `sections`. Повторяющиеся якоря и ложные разделы, начинающиеся с таблицы, исключаются. Переход из поиска использует стабильный ID блока, прокручивает и временно подсвечивает абзац или таблицу; при наличии корректного родительского раздела он также отмечается в содержании.

## Известные ограничения

HTML не воспроизводит плавающие текстовые поля, WordArt, сложные формулы, колонтитулы и постраничную геометрию Word пиксель в пиксель. Для точной визуальной сверки остаются оригинал и `preview.pdf`. Технические формулировки и содержимое документов не редактировались.
